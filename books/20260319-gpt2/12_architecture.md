これまでの章では、GPT-2 を構成する個々の部品を順に見てきました。この章では全体の設計思想から各部品の役割を捉え直します。

簡単のため、KV キャッシュを除いた構造で説明します。

# GPT-2 の全体構造

GPT-2 の推論は、わずか数行のコードに集約されます。

```python
class GPT2:
    def __call__(self, input_ids):
        x = self.wte[input_ids] + self.wpe[np.arange(len(input_ids))]
        for block in self.blocks:
            x = block(x)
        x = self.ln_f(x)
        return x @ self.wte.T
```

これを上から読むと、GPT-2 は3つの段階で構成されていることがわかります。

1. **入力**: トークン ID を連続ベクトル空間に写像する (Embedding)
2. **変換**: ベクトル列を12回繰り返し変換する（Transformer Block × 12）
3. **出力**: ベクトルを語彙の確率分布に戻す（最終 LayerNorm + LM Head）

入力と出力は空間の変換（離散 ↔ 連続）であり、モデルの本質的な処理は中間の「変換」にあります。

# 残差ストリーム: 情報の幹線

Transformer Block の内部構造とコードを再掲します。

```text
Input ─────┐
  ↓        │
LayerNorm  │
  ↓        │
Attention  │
  ↓        │
(+) ←──────┘
  ├────────┐
  ↓        │
LayerNorm  │
  ↓        │
MLP        │
  ↓        │
(+) ←──────┘
  ↓
Output
```

```python
class TransformerBlock:
    def __call__(self, x):
        x = x + self.attn(self.ln_1(x))
        x = x + self.mlp (self.ln_2(x))
        return x
```

推論時の情報の流れに注目すれば、`x = x + f(x)` という残差接続の構造は、「`x` という幹線に `f(x)` が情報を書き込む」と読み替えられます。この幹線を**残差ストリーム（Residual Stream）** と呼びます。

残差ストリームの視点では、Attention や MLP は幹線を「書き換える」のではなく、幹線に情報を「追加する」だけです。各層は独立に残差ストリームを読み取り、自分の計算結果を加算します。この加算的な構造により、初期の情報 (Embedding) は最終層まで直接伝わります。また、学習時には、勾配が加算のパスをそのまま逆流できるため、層が深くなっても勾配消失が起きにくくなります。

# Transformer Block の役割

1つのブロック内で Attention と MLP が行うことを整理します。

## Attention: トークン間の情報交換

Attention は「他のトークンから情報を集める」処理です。各トークンが Query を出し、他のトークンの Key との類似度に応じて Value を重み付き平均します。

- **入力**: 各トークンが独立に持つベクトル
- **出力**: 文脈を反映したベクトル（他のトークンの情報が混入）

因果マスクにより、各トークンは自分より前のトークンしか参照できません。推論（生成）時には、最後のトークンだけが文全体を「見ている」位置であり、次の単語の予測に使われます。

## MLP: トークン内の特徴変換

MLP は「各トークンのベクトルを個別に変換する」処理です。768 次元から 3072 次元に拡大し、非線形変換 (GELU) を適用してから 768 次元に戻します。

- **入力**: 1つのトークンのベクトル（他のトークンは見ない）
- **出力**: 特徴が変換されたベクトル

Attention がトークン間の「横方向」の処理なら、MLP はトークン内の「縦方向」の処理です。

## なぜこの順序なのか

1. **Attention で文脈を取り込む**: まず周囲の情報を集める
2. **MLP で特徴を変換する**: 集めた情報を踏まえて、各トークンの表現を深める

この 2 ステップを 1 セットとして 12 回繰り返すことで、浅い文法的パターンから深い意味的パターンまで、段階的に抽出していきます。

# 12 層の意味

層を経るごとに Embedding とのコサイン類似度は急速に低下します（👉[09](09_residual)）。これは各層が異なる抽象度の特徴を担っていることを示唆しています。

一般的に、Transformer の各層は以下のような傾向を持つことが知られています。

- **浅い層（0〜3）**: 文法構造、品詞パターン、局所的な共起関係
- **中間層（4〜8）**: 意味的な関係、エンティティの同定、文脈の統合
- **深い層（9〜11）**: タスク固有の判断、最終的な予測に向けた情報の絞り込み

ただし、これは明確に分かれるわけではなく、複数の層が協調して 1 つの機能を実現しています。

# LayerNorm の配置

GPT-2 では LayerNorm が 3 箇所に配置されています。

```python
# ブロック内（Pre-LayerNorm）
x = x + self.attn(self.ln_1(x))   # Attention の前
x = x + self.mlp (self.ln_2(x))   # MLP の前

# ブロック外
x = self.ln_f(x)                  # LM Head の前
```

ブロック内の LayerNorm は Attention/MLP への入力を正規化し、計算を安定させます。残差ストリーム上の `x` 自体は正規化されず、情報がそのまま保持されます。

最終 LayerNorm（ln_f）は、12層で蓄積されたスケールの増大（👉[09](09_residual): 標準偏差 0.22 → 14.2）を出力の前に整える役割です。

# GPT-2 から現代の LLM へ

GPT-2 のアーキテクチャは、現代の LLM の基本形です。大規模モデルとの本質的な違いはパラメータ数と学習データの規模であり、基本構造は驚くほど変わっていません。

主な発展は以下の点です。なお、商用 LLM の詳細は非公開のため、「現代の LLM」は Llama 等のオープンウェイトモデルに基づく一般的な傾向です。

| | GPT-2 | 現代の LLM |
|---|---|---|
| パラメータ数 | 124M | 数十B〜数百B |
| 層数 | 12 | 数十〜百以上 |
| コンテキスト長 | 1024 | 100K〜2M |
| 位置埋め込み | 学習済み絶対位置 | RoPE 等の相対位置 |
| 正規化 | LayerNorm | RMSNorm |
| 活性化関数 | GELU | SwiGLU |
| Attention | 標準 MHA | GQA / MLA |

Attention の行に注目すると、GQA（Grouped Query Attention）や MLA（Multi-head Latent Attention）は、KV キャッシュ（👉[11](11_kv_cache)）のメモリ削減を主な動機としています。標準の MHA (Multi-head Attention) では全ヘッドが独立に K, V を持つため、層数やコンテキスト長の増大に伴いキャッシュが膨大になります。GQA は複数のヘッドで K, V を共有することで、推論時のメモリ使用量を抑えます。

構造の違いは効率やスケーラビリティの改善であり、「残差ストリーム上で Attention と MLP を繰り返す」という根本的な設計は共通しています。GPT-2 のコンパクトな実装で理解した原理は、そのまま現代の LLM にも応用できます。
