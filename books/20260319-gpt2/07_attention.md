# Attention

Transformer の原論文のタイトルは "Attention Is All You Need"（Vaswani et al., 2017）であり、Attention こそが Transformer の中核です。従来のモデルで使われていた再帰構造や畳み込みを排し、Attention だけで系列全体の依存関係を捉えられることを示しました。

# Q, K, V（Query, Key, Value）

次の単語を予測するには、現在のトークンだけでなく文脈中の他のトークンの情報も必要です。どのトークンがどれだけ関連するかを判断し、必要な情報を集めてくる仕組みが Attention です。

この「関連するトークンを探して情報を集める」処理は、検索の比喩によって 3 つの役割が割り当てられます。

- **Q (Query)**: 「何を探しているか」（検索クエリ）
- **K (Key)**: 「自分は何を持っているか」（検索インデックス）
- **V (Value)**: 「実際の情報」（検索結果）

各トークンは、Q, K, V の 3 つのベクトルに変換されます。

GPT-2 では各トークンを 768 次元のベクトルで表し、それを 12 個の**ヘッド**に分割します（各ヘッド 64 次元）。ヘッドとは、独立に Attention を計算する並列の処理単位です。複数のヘッドがそれぞれ異なる観点で文脈を捉えることで、多面的な情報を取り込めます。この仕組みを **Multi-Head Attention** と呼びます。

Q, K, V の生成と出力射影に使う重みとバイアスは safetensors ファイルから読み込まれます。ヘッド数は `config.json` から取得します。GPT-2 では 12 です。

```python
Attention(
    n_head=config["n_head"],                  # 12
    w_qkv=weights["h.0.attn.c_attn.weight"],  # (768, 2304) Q,K,V 結合
    b_qkv=weights["h.0.attn.c_attn.bias"],    # (2304,)
    w_out=weights["h.0.attn.c_proj.weight"],  # (768, 768) 出力射影
    b_out=weights["h.0.attn.c_proj.bias"],    # (768,)
)
```

`c_attn` の重み行列は 768×2304 で、2304 = 768×3（Q, K, V の3つ分）です。1 回の行列積で Q, K, V をまとめて計算してから 3 分割します。この `x @ w_qkv` は成分を混合させる線形変換（768 次元から 2304 次元への写像）です。

```python
class Attention:
    def __call__(self, x):
        # 768次元の入力に (768, 2304) の重み行列を掛け、Q,K,V を一括計算
        qkv = x @ self.w_qkv + self.b_qkv
        # 2304次元を3等分して Q, K, V（各768次元）に分離
        q, k, v = np.split(qkv, 3, axis=-1)
```

続いて、Q, K, V をヘッドごとに分割します。768 次元を 12 ヘッド × 64 次元に reshape し、transpose でヘッド軸を先頭に移動します。

```python
        # Q, K, V をヘッドに分割: (seq_len, 768) → (12, seq_len, 64)
        def split_heads(tensor):
            tensor = tensor.reshape(seq_len, self.n_head, head_size)
            return tensor.transpose(1, 0, 2)
        q, k, v = map(split_heads, [q, k, v])
```

## 設計の動機：なぜ Q, K, V に分けるのか？

入力ベクトルをそのまま使うと、検索する側と検索される側の役割を区別できません。異なる投影行列を用意することで、同じ単語でも「文脈を探しに行く時の表現」と「文脈のヒントを与える時の表現」を使い分けられるようになります。

# スコア計算とスケーリング

検索エンジンにクエリを入力すると、各ページのインデックスとの一致度でランキングされるように、各トークンの Q（クエリ）をすべてのトークンの K（インデックス）に突き合わせて、関連度のスコア行列を作ります。この突き合わせには内積を使います。内積はベクトルが同じ方向を向いているほど値が大きくなるため、関連性の高いトークン同士ほどスコアが高くなります。

ただし、次元数が大きいと内積の値が非常に大きくなり、特定のトークンのスコアが過剰に高くなってしまいます。これを防ぐために、次元数 $\sqrt{d_k}$ で割ることでスケーリングします。

```python
# q: (head, seq_len, d_k), k: (head, seq_len, d_k)
# transpose で k の最後の2軸を入れ替え → (head, d_k, seq_len)
# q @ k^T: (head, seq_len, seq_len) → ヘッドごとに全トークン間のスコア行列
scores = q @ k.transpose(0, 2, 1) / np.sqrt(d_k)
```

`@` 演算子は最後の 2 軸で行列積を計算し、先頭の軸（head）は独立した行列積のインデックスです。ここでの `q @ k^T` は線形変換ではなく、各トークンのベクトル同士の内積を総当たりで一括計算しています。結果は変換されたベクトルではなく、スカラー値（スコア）の表になります。つまり全ヘッドのスコア行列を一括計算しています。

# Softmax

スコアを確率分布に変換するため、各スコアの指数 `exp(x)` を取り、合計で割ることで全体の和が 1 になるよう正規化します。これを **Softmax** と呼びます。最もスコアが高い要素に高い確率が割り当てられ、スコアの差が大きいほど確率の偏りも大きくなります。

$$
\text{Softmax}(x_i) = \frac{\exp(x_i)}{\sum_j \exp(x_j)}
$$

※ 分母 $\sum_j \exp(x_j)$ は統計力学の分配関数に対応します。

単純に実装すると `np.exp(x) / np.sum(np.exp(x))` となりますが、`x` の値が大きいと `exp(x)` がオーバーフローしてしまうため、最大値を引くことで回避します。最大値を引いても分子・分母に同じ定数がかかるため、結果は変わりません。

また、`x` は複数のベクトルを含んでいるため、`axis` と `keepdims` を指定することで、正しい次元で計算できるようにします。（詳細は後述）

```python
def softmax(x):
    # 最大値を引いてからexp（数値安定化）
    exp_x = np.exp(x - np.max(x, axis=-1, keepdims=True))
    # 合計で割って確率分布に変換
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)
```

前節のスケーリングはこの Softmax の性質と密接に関係しています。スケーリングなしでスコアが大きくなりすぎると、Softmax の出力がほぼ one-hot（1箇所だけ 1 で他は 0）になり、他のトークンからの情報を取り込めなくなります。

## 補足：axis と keepdims の役割

2次元配列を例に動作を確認します。

```python
>>> a = np.array([[1, 2, 3],
...               [4, 5, 6]])
>>> a.shape
(2, 3)
```

`axis` は操作する軸を指定します。`axis=0` は行方向（縦）、`axis=1`（= `axis=-1`）は列方向（横）に集約します。

```python
>>> np.max(a, axis=0)          # 各列の最大値
array([4, 5, 6])               # shape: (3,)
>>> np.max(a, axis=1)          # 各行の最大値
array([3, 6])                  # shape: (2,)
```

`keepdims=True` を付けると、集約した軸が大きさ 1 の次元として残ります。これにより元の配列との引き算などでブロードキャスト（shape が異なる配列同士の演算で、大きさ 1 の軸を自動的に拡張して要素ごとに計算する仕組み）が正しく動きます。

```python
>>> m = np.max(a, axis=1, keepdims=True)
>>> m
array([[3],   # shape: (2, 1)
       [6]])
>>> a - m     # (2, 3) - (2, 1) → ブロードキャスト可能
array([[-2, -1,  0],
       [-2, -1,  0]])
```

`keepdims=False`（デフォルト）だと次元が潰れるため、ブロードキャストができなくなります。

```python
>>> m = np.max(a, axis=1)  # keepdims=False と同じ
>>> m
array([3, 6])  # shape: (2,)
>>> a - m
ValueError: shapes (2,3) and (2,) not aligned
```

# 因果マスキング（Causal Masking）

GPT-2 は「次の単語を予測する」モデルであるため、未来の単語を見てはいけません。参照の可否を図にすると、可能な範囲は下三角行列の形になります。4トークンの場合を示します。

|          | tok0 | tok1 | tok2 | tok3 |
|----------|------|------|------|------|
| **tok0** | OK   | NG   | NG   | NG   |
| **tok1** | OK   | OK   | NG   | NG   |
| **tok2** | OK   | OK   | OK   | NG   |
| **tok3** | OK   | OK   | OK   | OK   |

※ 「縦の見出し」から「横の見出し」を見た時の可否を表しています。例えば tok1 は tok0, tok1 を参照できますが、tok2, tok3 は参照できません。

各トークンが自分自身と過去のトークンだけを参照できるようにするには、未来のスコアを $-10^{10}$（実質 $-\infty$）で上書きし、Softmax 適用後の確率を 0 にします。`np.tril` で下三角行列を作り、参照不可範囲のスコアを書き換えます。

```python
# 下三角行列を生成（対角線と左下が1、右上が0）
mask = np.tril(np.ones((seq_len, seq_len)))
# mask が 0 の位置（未来）を -1e10 で上書き
scores = np.where(mask == 0, -1e10, scores)
# Softmax で確率に変換（-1e10 → ほぼ 0）
probs = softmax(scores)  # 注目度行列
```

入力 "The capital of France is" での注目度行列 `probs` から Head 0 の部分を抜粋します。

|          |     'The'|' capital'|     ' of'| ' France'|     ' is'|
|----------|---------:|---------:|---------:|---------:|---------:|
|     'The'|    1.0000|          |          |          |          |
|' capital'|    0.8464|    0.1536|          |          |          |
|     ' of'|    0.7014|    0.2237|    0.0749|          |          |
| ' France'|    0.6077|    0.1788|    0.1439|    0.0696|          |
|     ' is'|    0.6099|    0.1519|    0.0626|    0.0945|    0.0812|

最初のトークン 'The' は自分だけに 100% 注目し、最後のトークン ' is' はすべてのトークンを参照できます。このように最後のトークンに情報が集約されるのが因果マスキングの特徴です。

## 補足：Attention の種類と他のモデル

GPT-2 の Attention は、同じ系列内のトークン同士で注目し合うため **Self-Attention** と呼ばれます。因果マスキングにより未来を見ないようにしていますが、すべてのモデルがこの制約を持つわけではありません。

Transformer の原論文（Vaswani et al., 2017）は翻訳タスクを対象とし、**Encoder**（入力を双方向に参照して表現を作る）と **Decoder**（過去のみを参照して出力を生成する）の 2 つの部分から構成されています。Decoder には Self-Attention に加えて、Encoder の出力を参照する **Cross-Attention** もあります。Cross-Attention では Q が Decoder 側、K と V が Encoder 側から来ることで、「翻訳元のどの単語に注目するか」を決めます。

GPT-2 は Decoder のみの構成なので Self-Attention だけで動いています。一方 BERT（Devlin et al., 2019）は Encoder のみを使い、同じく Self-Attention ですが因果マスクなしで文全体を双方向に参照します。入力の一部のトークンを `[MASK]` に置き換え、その元の単語を当てる学習方式（Masked Language Model）を取ります。BERT は主に文章の理解（分類や質問応答）に使われますが、マスクした位置を繰り返し予測して埋めることで文章を生成する試みもあります。テキストの拡散モデルもこの系譜にあり、GPT のような左から右への逐次生成とは異なるアプローチです。

# Value の重み付き平均

ここまでの処理で、各トークンが他のトークンにどれだけ注目すべきかの確率分布（probs）が得られました。この確率分布を重みとして、V（各トークンが持つ「実際の情報」）の加重平均を取ります。

```python
# probs: (head, seq_len, seq_len) — 各トークンの注目度分布
# v:     (head, seq_len, d_k)     — 各トークンの情報ベクトル
# out:   (head, seq_len, d_k)     — 文脈を反映した新しいベクトル
out = probs @ v
```

この行列積も、線形変換ではなく加重平均の一括計算です。probs の各行が確率分布（合計 1）なので、V の行ベクトルを確率に応じて混合した結果が得られます。

具体的に見るため、注目度行列 `probs` で ' is' の行を再掲します。

|     |     'The'|' capital'|     ' of'| ' France'|     ' is'|
|-----|---------:|---------:|---------:|---------:|---------:|
|' is'|    0.6099|    0.1519|    0.0626|    0.0945|    0.0812|

' is' の出力ベクトルは、各トークンの V をこの注目度で混合したものになります。'The' の V に 0.61、' capital' の V に 0.15 を掛けて足し合わせるというように、注目度の高いトークンほど多くの情報が流れ込みます。

実際の計算結果（Head 0、先頭 5 次元）：

```text
各トークンの V:
         'The': [-0.0131, -0.0145, +0.1269, -0.0221, -0.2082, ...]
    ' capital': [+0.0023, +0.1138, +0.1138, +0.3719, +0.1921, ...]
         ' of': [-0.0043, -0.1001, -0.0527, -0.0737, -0.0805, ...]
     ' France': [-0.1402, -0.0411, -0.2345, +0.1896, +0.0455, ...]
         ' is': [+0.2624, -0.2693, +0.3042, -0.1792, -0.1881, ...]

加重平均の結果（' is' の出力）:
         ' is': [+0.0002, -0.0235, +0.0939, +0.0418, -0.1138, ...]
```

たとえば先頭の次元では、各トークンの V の値に注目度を掛けて足し合わせることで、出力の +0.0002 が得られます。

```text
-0.0131 * 0.6099 = -0.0079692  ('The')
+0.0023 * 0.1519 = +0.0003562  (' capital')
-0.0043 * 0.0626 = -0.0002718  (' of')
-0.1402 * 0.0945 = -0.0132526  (' France')
+0.2624 * 0.0812 = +0.0213002  (' is')
                計 +0.0002
```

これが Attention の核心です。各トークンが、自分にとって重要な他のトークンから情報を集めて、文脈を反映した新しいベクトルを得ます。入力時は独立だったトークンのベクトルが、この処理を通じて互いの情報を取り込み、「文脈を理解した」表現に変わります。

# Multi-Head の統合

ここまでの処理は全12ヘッドで同時に行われています。これまで Head 0 を例に見てきましたが、各ヘッドはそれぞれ異なる「注目パターン」を学習しています。

最後のトークン ' is' の注目先（上位3）：

| Head | 1位 | 2位 | 3位 |
|------|-----|-----|-----|
| Head 0 | 'The'(0.610) | ' capital'(0.152) | ' France'(0.094) |
| Head 1 | ' is'(0.978) | ' of'(0.015) | 'The'(0.003) |
| Head 2 | 'The'(0.469) | ' of'(0.209) | ' is'(0.181) |
| Head 3 | ' is'(0.798) | 'The'(0.106) | ' of'(0.067) |

Head 0 は 'The' や ' capital' など文の主要な単語に分散して注目し、Head 1 は自分自身に集中するなど、各ヘッドが異なる関係性を捉えています。

各ヘッドの結果（64 次元 × 12 ヘッド）を結合して元の 768 次元に戻し、出力射影行列を掛けます。

```python
# ヘッドの結合: (12, seq_len, 64) → (seq_len, 12, 64) → (seq_len, 768)
out = out.transpose(1, 0, 2).reshape(seq_len, embed_dim)
# 出力射影（アフィン変換）
return out @ self.w_out + self.b_out
```

この出力はまだ次のトークンを決定するものではありません。この後、残差接続と LayerNorm を経て MLP（フィードフォワード層）を通り、さらにこれを 12 層繰り返した最終出力から次のトークンが予測されるためです。

# まとめ

GPT-2 の Attention の処理は以下の流れで進みます。

1. 各トークンのベクトルから Q, K, V を生成し、ヘッドに分割する
2. Q と K の内積でスコアを計算する
3. 未来のトークンをマスクで遮断する
4. Softmax でスコアを確率分布に変換する
5. 確率に応じて V の加重平均を計算する
6. 全ヘッドの結果を結合し、出力射影を計算する
