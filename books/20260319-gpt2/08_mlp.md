# MLP の役割

Attention はトークン間で情報を集約しますが、集めた情報を「解釈」する処理は行いません。MLP がその役割を担います。768 次元のベクトルを一度 4 倍の 3072 次元に拡張し、非線形変換してから元の 768 次元に圧縮します。非線形変換がないと、どれだけ層を重ねても全体が単なる線形変換に潰れてしまうため、非線形性の導入が不可欠です。

MLP の重みとバイアスは safetensors ファイルから読み込まれます。

```python
MLP(
    w_fc=weights["h.0.mlp.c_fc.weight"],      # (768, 3072) 拡張
    b_fc=weights["h.0.mlp.c_fc.bias"],        # (3072,)
    w_proj=weights["h.0.mlp.c_proj.weight"],  # (3072, 768) 射影
    b_proj=weights["h.0.mlp.c_proj.bias"],    # (768,)
)
```

MLP の処理は 2 つのアフィン変換と、その間の活性化関数 GELU で構成されます。（`seq_len` はトークン数）

```python
class MLP:
    def __call__(self, x):
        # (seq_len, 768) → (seq_len, 3072) 拡張
        i = x @ self.w_fc + self.b_fc
        # (seq_len, 3072) 非線形変換
        a = gelu(i)
        # (seq_len, 3072) → (seq_len, 768) 射影
        return a @ self.w_proj + self.b_proj
```

ここで `i` が 3072 次元の**中間層**です。入力の 768 次元より 4 倍広い空間で特徴を展開し、非線形変換で選別してから元の次元に戻します。

# 拡張: 768 → 3072

最初のアフィン変換 `i = x @ self.w_fc + self.b_fc` で、768 次元の入力を 3072 次元の中間層に拡張します。

入力 "The capital of France is" の変換結果を示します。

|   トークン   | 正の成分  | 負の成分  |
|--------------|-----------|-----------|
| `'The'`      |  210/3072 | 2862/3072 |
| `' capital'` |  419/3072 | 2653/3072 |
| `' of'`      |  276/3072 | 2796/3072 |
| `' France'`  |  651/3072 | 2421/3072 |
| `' is'`      |  284/3072 | 2788/3072 |

中間層の大部分が負の値になっています。この偏りが次に説明する活性化関数 GELU で重要な役割を果たします。

## 設計の動機：なぜ 4 倍に拡張するのか？

MLP は高次元化と非線形化の組み合わせで複雑なパターンを処理します。中間層を 768 の 4 倍の 3072 次元にすることで、多くの特徴を同時に表現できる空間を確保します。続く非線形変換によって、線形では分離できないパターンも区別可能になります。

同様の例として、XOR（排他的論理和）を考えます。XOR の出力が 0 になる点 (0,0), (1,1) と、1 になる点 (0,1), (1,0) は、2 次元平面上でどのような直線を引いても分離できません。そこで、非線形な特徴 $x_1 x_2$ を加えて 3 次元に拡張します。

$$
(x_1, x_2) \mapsto (x_1, x_2, x_1 x_2)
$$

これによって、XOR=0 の点は (0,0,0), (1,1,1)、XOR=1 の点は (0,1,0), (1,0,0) となり、以下の平面で線形分離できるようになります。

$$
x_1 + x_2 - 2x_3 = \frac{1}{2}
$$

# 活性化関数 GELU

「活性化関数」は生物の神経細胞（ニューロン）からの類推です。ニューロンは入力が閾値を超えると発火（活性化）し、信号を次に伝えます。活性化関数はこれを模倣し、入力のうち重要な特徴を通過させ、それ以外を抑制する非線形変換です。

## ReLU から GELU へ

ニューラルネットワークの活性化関数として広く使われてきた **ReLU** (Rectified Linear Unit) は、正の値をそのまま通し、負の値をゼロにする単純な関数です。

$$
\text{ReLU}(x) = \max(0, x)
$$

ReLU は計算が高速で効果的ですが、x=0 で微分不可能（角がある）になります。GPT-2 はこれを滑らかにした **GELU** (Gaussian Error Linear Unit) を採用しています。

$$
\text{GELU}(x) \approx 0.5 \times x \times \left(1 + \tanh\left(\sqrt{\frac{2}{\pi}} \times (x + 0.044715 \times x^3)\right)\right)
$$

```python
def gelu(x):
    return 0.5 * x * (1 + np.tanh(np.sqrt(2 / np.pi) * (x + 0.044715 * x ** 3)))
```

ReLU と同様に、正の値はほぼそのまま通過し、負の値は大きく抑制されます。ReLU との違いは境界付近が滑らかに遷移することで、学習時の勾配が安定しやすくなります。

## GELU の最小値と実質的なゼロ

ReLU は負の入力を完全にゼロにしますが、GELU は滑らかにした分だけ小さな負の値が残ります。数値的に求めると、GELU の出力の最小値は約 **-0.17**（x ≈ -0.75 付近）です。どんな負の入力に対しても出力は -0.17 より小さくなりません。

この -0.17 という値は、ReLU の折れ曲がりを滑らかにした際の「はみ出し」です。ReLU であればゼロになる部分なので、実質的にゼロとみなせます。つまり GELU は ReLU とほぼ同じフィルタリング効果を持ちつつ、微分可能という利点を備えています。

代表的な入力値で確認します：

| 入力 | GELU 出力 | 備考 |
|-----:|----------:|------|
| -2.0 | -0.0454 | ほぼ 0 に抑制 |
| -1.0 | -0.1588 | 最小値 -0.17 に近い |
| -0.5 | -0.1543 | 同上 |
|  0.0 |  0.0000 | |
|  0.5 |  0.3457 | 入力より少し小さい |
|  1.0 |  0.8412 | 入力に近い |
|  2.0 |  1.9546 | ほぼそのまま通過 |

# 射影: 3072 → 768

GELU で選別された 3072 次元のベクトルに (3072, 768) の重み行列を掛けて、元の 768 次元に戻します。

```python
# (seq_len, 3072) → (seq_len, 768)
output = a @ self.w_proj + self.b_proj
```

この射影は、GELU で活性化された次元の情報を 768 次元に集約する処理です。ゼロに近い次元は結果にほとんど寄与しないため、実質的に活性化された次元だけが出力に反映されます。

# MLP によるベクトルの変化

MLP 適用前後のベクトルを比較すると、大きな変化が起きていることが分かります。（std は標準偏差）

|   トークン   | 入力std  | 出力std  | コサイン類似度 |
|--------------|----------|----------|----------------|
| `'The'`      |   0.2058 |   4.4876 |         0.3055 |
| `' capital'` |   0.1636 |   1.0964 |         0.3250 |
| `' of'`      |   0.1539 |   1.0827 |         0.2095 |
| `' France'`  |   0.1494 |   1.2973 |         0.3210 |
| `' is'`      |   0.1472 |   1.1864 |         0.1841 |

入力と出力のコサイン類似度が 0.2〜0.3 と低く、MLP がベクトルの方向を大きく変えていることが分かります。標準偏差も大幅に増加しており、MLP が情報を積極的に書き込んでいることが分かります。特に 'The' は標準偏差が 0.21 から 4.49 と約 22 倍に増加しています。

ただし、この出力はそのまま次の層に渡されるのではなく、次回に説明する残差接続（👉[09](09_residual)）によって元のベクトルに加算されます。MLP の大きな変化も、残差接続で元のベクトルに足し合わされることで緩和され、情報の上書きではなく「追記」として機能します。

## 補足：Attention と MLP の分業

Transformer Block 内の Attention と MLP は異なる役割を担っています。

- **Attention**: トークン間の情報伝達。「どの単語を参照するか」を決めて情報を集める（横方向の処理）
- **MLP**: トークン単体の特徴変換。集めた情報を解釈し、次の層に向けて意味を加工する（縦方向の処理）

この 2 つが交互に繰り返されることで、トークンのベクトルは徐々に文脈を取り込み、「次の単語を予測する」ために必要な表現へと変化していきます。

# まとめ

GPT-2 の MLP の処理は以下の流れで進みます。

1. 768 次元のベクトルに重み行列を掛けて 3072 次元に拡張する
2. 活性化関数 GELU でトークンごとに重要な特徴を選別する
3. 3072 次元のベクトルに重み行列を掛けて 768 次元に射影する

構造は単純ですが、3072 次元への拡張と GELU による選別により、各トークンに固有の特徴パターンを書き込む重要な役割を果たしています。
