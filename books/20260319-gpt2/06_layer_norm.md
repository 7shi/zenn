# LayerNorm の仕組み

Transformer Block 内の処理では `x = x + f(x)` という形でベクトルの加算が行われます。

※ この加算を残差接続と呼びます。詳細は別途説明します（👉[09](09_residual)）。

残差接続によって各層の出力が積み重なると、ベクトルのスケールはトークンごと・層ごとにばらつきます。このばらつきを放置すると、後続の Attention（👉[07](07_attention)）のスコア計算（内積）が特定の次元に支配されやすくなり、計算が不安定になります。LayerNorm は各処理の直前でスケールを揃えることで、この問題を防ぎます。

具体的には、各トークンのベクトル（768次元）を**平均 0・分散 1** に正規化してから、学習によって調整されたパラメーター **γ**（スケール）と **β**（シフト）を適用します。γ/β という記号は BatchNorm の原論文（Ioffe & Szegedy, 2015）に由来する慣習です。

γ と β は safetensors ファイルから読み込まれます。

```python
LayerNorm(
    g=weights["h.0.ln_1.weight"],  # γ (768,)
    b=weights["h.0.ln_1.bias"],    # β (768,)
)
```

LayerNorm では、各トークンのベクトル全成分の平均と分散を計算して正規化し、γ/β を成分ごとに適用して学習済みのスケール感を与えます。

```python
class LayerNorm:
    def __call__(self, x, eps=1e-5):
        # 768次元の全成分についてトークンごとの平均を計算
        mean = np.mean(x, axis=-1, keepdims=True)
        # 同様に分散を計算（ばらつきの指標）
        variance = np.var(x, axis=-1, keepdims=True)
        # 平均を引いて標準偏差で割り、平均0・分散1に正規化
        x_norm = (x - mean) / np.sqrt(variance + eps)
        # γ で次元ごとにスケールし、β でシフト（アダマール積）
        return self.g * x_norm + self.b
```

`self.g * x_norm` は行列積ではなく**アダマール積**（成分ごとの積）です。γ と β はそれぞれ 768 次元のベクトルで、各次元を独立してスケール・シフトします。次元間の情報は混合しないため、「次元ごとのアフィン変換」とも呼ばれます。

## LayerNorm の効果

Embedding 直後のベクトルは、トークンごとに分散がばらついています。正規化によって、γ/β 適用前の段階ではすべてのトークンが平均 0・分散 1 に統一されますが、その後のアフィン変換によって最終的な値は変化します。

|   トークン   | 平均(前) | 分散(前) | 平均(後) | 分散(後) |
|--------------|---------:|---------:|---------:|---------:|
| `'The'`      |  -0.0068 |   0.1336 |  -0.0050 |   0.0050 |
| `' capital'` |  -0.0016 |   0.0508 |  -0.0072 |   0.0130 |
| `' of'`      |  -0.0042 |   0.0363 |  -0.0048 |   0.0080 |
| `' France'`  |  -0.0024 |   0.0399 |  -0.0051 |   0.0119 |
| `' is'`      |  -0.0043 |   0.0317 |  -0.0043 |   0.0077 |

## γ と β の役割

正規化で平均 0・分散 1 に揃えると、768 次元すべてが同じスケールになります。しかし、モデルにとっては「この次元は大きく、あの次元は小さく」という差異こそが重要な情報です。γ（次元ごとの倍率）と β（次元ごとのオフセット）を学習することで、正規化の安定性を保ちつつ、必要なスケールの違いを復元します。

```
γ: 平均= 0.1804, 分散=0.0017, 最小= 0.0419, 最大= 0.2527
β: 平均=-0.0066, 分散=0.0013, 最小=-0.2589, 最大= 0.2019
```

γ の平均が約 0.18 であることから、正規化後のベクトルは元の約 1/5 のスケールに縮小されています。β はほぼ 0 付近で、大きなシフトは加えていません。全体として、後続の Attention（👉[07](07_attention)）や MLP（👉[08](08_mlp)）に渡す値の範囲を抑制する方向に学習されています。

## 通常の正規化との違い

LayerNorm は「ベクトルの正規化」（長さを 1 にする L2 正規化）とは異なります。L2 正規化はベクトルの方向を変えないため、コサイン類似度には影響しません。一方 LayerNorm は、平均を引く操作と γ/β による成分ごとのアフィン変換を含むため、ベクトルの方向自体を変化させます。

# GPT-2 での使用箇所

GPT-2 では LayerNorm を合計 25 回使います。各 Transformer Block に `ln_1`（Attention 前）と `ln_2`（MLP 前）の 2 つが入るため 12 ブロック × 2 = 24 回、最後に `ln_f`（LM Head 前）が 1 回加わります。
